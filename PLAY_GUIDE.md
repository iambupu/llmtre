# TRE 最小可玩版本游玩指南

本指南面向想直接试玩当前 Web Demo 的玩家。按本文顺序操作，可以完成一次新会话创建、回合输入、场景查看、记忆查看和重启后继续游玩。沙盒并入/回滚属于 A1 实验功能，普通试玩不要点击。

## 1. 游玩前准备

### 1.1 安装依赖

项目要求 Python `3.14+`。

```bash
pip install -r requirements.txt
```

### 1.2 准备本地模型

当前验证过的默认配置使用 Ollama：

- LLM：`qwen3:8b`
- Embedding：`bge-m3`

如果使用其他模型，需要同步修改：

- RAG 模型：`config/rag_config.yml`
- Agent 叙事模型：`config/agent_model_config.yml`

### 1.3 初始化数据库、知识库和 MOD

首次运行推荐按顺序执行：

```bash
python state/tools/db_initializer.py
python tools/doc_importer.py docs/ --group core --sync
python tools/mod_manager.py scan
```

说明：

- `docs/` 默认不会提交到 Git；需要先放入规则书、世界观或设定文档，再导入知识库。
- `python app.py` 启动时也会尝试自动补齐 SQLite 和向量索引；如果自动索引失败，请回到本节手动执行初始化命令。

## 2. 启动游戏

```bash
python app.py
```

启动后在浏览器打开：

```text
http://localhost:5000/play
```

页面加载后，回合记录区会显示“页面已加载，可创建或加载会话。”

## 3. 页面区域说明

页面主要分为四块：

- 顶部栏：填写角色 ID、创建新会话、加载已有会话、重置会话、开关调试面板。
- 左侧主区域：显示当前场景、回合记录、建议行动和输入框。
- 右侧状态栏：显示角色 HP/MP、背包、当前任务和记忆摘要。
- 调试面板：显示状态快照、Agent 状态、日志、Trace、内存和输出配置。

玩家日常游玩主要使用：

- `新会话`：创建一次新冒险。
- `发送`：提交当前输入。
- `历史`：查看回合历史。
- `记忆` / `刷新记忆`：查看或更新近期剧情摘要。
- `并入主线` / `回滚沙盒`：A1 实验沙盒按钮。只有明确创建或进入沙盒分支后才能使用，普通试玩不要点击。

## 4. 第一局推荐流程

1. 确认顶部“角色”为 `player_01`。
2. 点击 `新会话`。
3. 等待 GM 开场叙事显示在“回合记录”中。
4. 阅读“当前场景”，重点看地点描述、出口、可见对象和建议行动。
5. 点击页面给出的建议行动，或在输入框输入 `观察周围` 后点击 `发送`。
6. 继续输入 3 到 5 个明确行动，例如：

```text
观察周围
检查背包
前往森林
攻击地精
和地精说话
使用药水
```

7. 点击 `历史` 查看已发生的回合。
8. 点击 `记忆` 查看近期剧情摘要。
9. 关闭并重新启动 `python app.py` 后，可把顶部“会话”输入框填回原 `session_id`，点击 `加载` 继续游玩。

## 5. 如何输入行动

当前 NLU 以规则和关键词为主，输入越明确越稳定。

| 想做的事 | 推荐输入 |
| :--- | :--- |
| 查看环境 | `观察周围`、`环顾四周` |
| 检查状态 | `检查背包`、`查看当前场景` |
| 移动 | `前往森林`、`移动到营地`、`继续前进` |
| 对话 | `和地精说话`、`与旅行者对话` |
| 互动 | `调查营地`、`尝试互动` |
| 使用物品 | `使用药水`、`喝下药水` |
| 休息等待 | `休息`、`等待一会` |
| 攻击 | `攻击地精` |

建议避免过短或缺目标的输入，例如 `过去`、`看看`、`弄一下`。如果系统认为行动不够明确，会返回澄清问题；按问题补充目标或方向即可。

## 6. 场景、记忆与建议行动

“当前场景”来自后端返回的 `SceneSnapshot`，会随创建会话、加载会话和提交回合刷新。

你需要重点关注：

- 当前位置：当前地点名称与描述。
- 出口：可以移动到哪里。
- 可见对象：当前可见 NPC、物品或任务。
- 建议行动：页面可点击的下一步行动。
- 最近记忆：系统记录的近期剧情摘要。

建议行动只是快捷输入，不是唯一可选项；你也可以直接在输入框输入其他明确行动。

## 7. 输出方式

默认使用“流式输出”。提交回合后，状态栏会依次显示理解意图、校验动作、执行结算和生成叙事等阶段，GM 文本会逐段出现。

如果流式输出异常，可以在右侧调试面板打开 `配置` 页签，把“输出方式”切换为 `普通输出` 后再试同一条输入。

## 8. 沙盒并入与回滚（A1 实验功能）

当前页面直接暴露两个沙盒按钮，但新建普通会话时 `sandbox_mode` 默认为 `false`，页面也还没有完整的“进入沙盒分支”玩家流程。

因此普通试玩时不要点击：

- `并入主线`
- `回滚沙盒`

只有在你已经通过测试流程或后端接口明确创建/进入沙盒分支，并确认 Shadow 状态中存在待处理变化后，才可以使用：

- `并入主线`：把当前 Shadow 分支变化合并到 Active 主线状态。
- `回滚沙盒`：放弃当前 Shadow 分支变化，恢复到 Active 主线状态。

如果不确定当前会话是否处于沙盒分支，保持正常输入行动即可，不要用这两个按钮做普通保存或撤销。

## 9. 重置与继续会话

- `重置`：清空当前会话的回合与记忆，把 `current_turn_id` 重置为 `0`。
- `加载`：输入已有 `session_id` 后恢复该会话。

会话数据保存在 SQLite 中。只要没有删除或重建 `state/core_data/tre_state.db`，重启 Flask 后仍可加载旧会话。

## 10. 常见问题

### 页面打不开

确认服务正在运行：

```bash
python app.py
```

然后访问：

```text
http://localhost:5000/play
```

### 创建会话或回合很慢

本地模型首次加载可能较慢。先确认 Ollama 正在运行，且 `qwen3:8b` 与 `bge-m3` 可用。

如果只想验证纯确定性链路，需要同时关闭 NLU 和 GM 的模型调用。仅关闭 GM 只会切换叙事模板，NLU 仍可能走 LLM 兜底。

`config/agent_model_config.yml` 示例：

```yaml
bindings:
  agents.nlu:
    enabled: false
    mode: "deterministic"
    llm_profile: null
  agents.gm:
    enabled: false
    mode: "deterministic"
    llm_profile: null
```

RAG 索引缺失时，启动会尝试初始化向量索引，embedding 仍可能依赖 Ollama。纯确定性验收时建议先准备好索引，或在 `config/main_loop_rules.json` 中临时关闭：

```json
{
  "rag": {
    "read_only_enabled": false,
    "auto_initialize": false
  }
}
```

### RAG 或知识库报错

重新导入并同步知识库：

```bash
python tools/doc_importer.py docs/ --group core --sync
```

同时确认 `config/rag_config.yml` 中的 embedding 模型可用。

### 系统要求澄清

这不是失败。说明输入缺少目标、方向或动作类型。按澄清问题补充更具体的行动即可，例如把 `过去` 改成 `前往森林`。

### 想确认运行日志是否正常

试玩过至少一个回合后执行：

```bash
python -m tools.logs.check_runtime_logs --since-minutes 15
```

输出中包含 `RUNTIME_LOG_CHECK_OK` 表示近期主循环、事件总线和外环日志证据完整。
