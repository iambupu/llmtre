# 智能体工作流执行与工具调用硬性操作规范

## 工具调用规范
1. 所有数值判定必须调用 `dice_roller.py`
2. 状态修改必须通过 `db_updater.py` 进行
3. 禁止直接修改 SQLite 数据库文件

## 数据流规范
1. 玩家输入 → NLU Agent → JSON 动作
2. 动作验证 → 规则检查 → 数值判定
3. 结果应用 → 状态更新 → 叙事反馈

## `.agent_context/` 使用规范
1. `MEMORY.md` 只记录长期叙事摘要、重大选择、世界态度变化和异常恢复记录。
2. 运行期只读加载 `MEMORY.md`，并挂载到 `SceneSnapshot.recent_memory`；禁止把它当作状态写入通道。
3. 当前 HP、MP、背包、位置、任务状态等结构化事实必须通过 SQLite/Pydantic 契约查询，不得从 `MEMORY.md` 推断。
4. 只有标题或占位注释的 `MEMORY.md` 视为空模板；写入真实条目后才会进入 Agent 上下文。

## 错误处理
- 所有异常必须记录到运行日志或专门运维日志；`MEMORY.md` 仅摘要记录会影响后续叙事解释的异常恢复事件
- 数据库操作失败必须回滚
- 无效输入必须返回友好错误提示
